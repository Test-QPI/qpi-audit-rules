Admissions QPI Knowledge Base (Authoritative)

All QPI evaluations must use these four execution bundles as the sole source of truth. Rules may not be inferred, recreated, or modified.

Version: KB v1.8  |  Scope: Interview and Follow-Up Calls

Bundle 1 — Execution and Evidence Contract

Invocation rule, evidence standard, execution order, hard gates, enumeration integrity, output schema, QPI total calculation.

  • qpi-execution-contract.html

Bundle 2 — Metadata, Stage Detection, Tasks and Constraints

Filename parsing, stage triggers, task enumeration, constraints, document timeline, progression definitions.

  • qpi-structure-detection.html

Bundle 3 — Scoring Engines

Progression scoring, binary compliance (Interview and Follow-Up), behavioral matrices (Interview and Follow-Up). Interview and Follow-Up rules are explicitly partitioned — use only the branch matching Call_Subtype.

  • qpi-scoring-engines.html

Bundle 4 — Tonality, Red Flags and Output Validation

Tonality definitions, count rules, empathy exclusions, Level A disqualifiers, red flags, assessment disclosure.

  • qpi-tonality-flags-output.html

Previous individual source files remain in this repository for reference only. The four bundles above are the active scoring sources.
