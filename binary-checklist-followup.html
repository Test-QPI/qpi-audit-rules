Binary Compliance Checklist – Follow-Up v1.1

Follow-Up Binary Compliance Checklist Definitions
Version 1.1 – Adds Blanket Prior Completion Confirmation Rule for Branch B

CHANGE LOG FROM v1.0
1) Added BLANKET PRIOR COMPLETION CONFIRMATION RULE — when agent's opening language
   explicitly establishes all upstream enrollment work is complete without item-by-item
   review, F2_1, F2_2, and F2_3 may each be scored 1 under this blanket path.
2) Extended PRIOR COMPLETION CREDIT RULE to explicitly include F2_1.
3) F2_4 (Assessment Re-engaged) is unaffected — explicit re-engagement is always required
   regardless of prior completion status.

PURPOSE
Define binary compliance items scored as 1 or 0 for Follow-Up calls using explicit transcript
evidence. This checklist supports a weighted Binary Compliance score (40 points) for
Follow-Up calls only.
Do NOT apply this checklist to Interview calls. Interview calls use the Interview Binary
Compliance Checklist.

APPLICABILITY RULE
This checklist applies ONLY when Call_Subtype = Follow-Up.
If Call_Subtype = Interview, stop. Use the Interview Binary Compliance Checklist instead.

BRANCH DETERMINATION (HARD RULE)
Before scoring any binary item, determine the correct branch using FAFSA_Validated_YN:

Branch A — FAFSA_Validated_YN = NO
The student has not yet submitted the FAFSA form. The agent's primary goal is FAFSA completion.
Score F1_1 through F1_4 only.

Branch B — FAFSA_Validated_YN = YES
The student has already submitted the FAFSA form. The agent's primary goal is document and
enrollment task completion.
Score F2_1 through F2_4 only.

Binary_Items_Applicable = 4 for all Follow-Up calls regardless of branch.
Assessment re-engagement (F1_4 and F2_4) is always applicable regardless of degree program.

--------------------------------------------------

GLOBAL SCORING RULES (ANTI-DRIFT)
- Score 1 = YES only when the agent performed the explicit action in the transcript.
- Score 0 if action is missing, unclear, implied, or performed only by a third party.
- No inference allowed.
- Default to 0 when uncertain.

PROOF-LOCK RULE (HARD RULE — APPLIES TO EVERY BINARY ITEM)
Before scoring any binary item as 1, you must be able to identify and internally quote the
exact agent or student words from this transcript that satisfy the criterion.
If you cannot identify specific verbatim language:
- The score must be 0
- Do not paraphrase
- Do not reconstruct
- Do not cite language from a different call
- Do not assume the action occurred because a related action occurred

FABRICATION PREVENTION
If you scored a binary item as 1 but the evidence you would cite does not appear word-for-word
in the transcript, correct the score to 0 before outputting. If the exact quote cannot be
located, the score is 0.

PRIOR COMPLETION CREDIT RULE
For items that involve status confirmation or task completion (F2_1, F2_2, and F2_3), a score
of 1 may be awarded if the agent confirmed the item was already completed prior to the call
AND acknowledged that confirmation explicitly before moving to remaining tasks.

Score 1 if the agent:
- Explicitly asked about or confirmed the status of the item during the call, AND
- Either walked the student through completion on-call OR confirmed prior completion and
  acknowledged it before pivoting

Score 0 if the agent did not address the item status at all or skipped it without any
confirmation.

BLANKET PRIOR COMPLETION CONFIRMATION RULE (NEW IN v1.1)
On Branch B calls, when the agent's opening language explicitly establishes that ALL upstream
enrollment work is complete — without reviewing each item individually — this blanket
confirmation satisfies the prior completion credit standard for F2_1, F2_2, and F2_3
simultaneously.

Score F2_1, F2_2, and F2_3 each as 1 under this rule ONLY if the agent's language:
- Explicitly positions the student as having completed all prior enrollment tasks before the
  call, AND
- Is spoken by the agent (not assumed from student-only language), AND
- Is unambiguous about the completion status of upstream enrollment work

Qualifying examples of blanket prior completion language:
- "What usually takes students days, you took care of before we even met"
- "You've already got your supplemental enrollment documents done — you're ahead of the game"
- "Everything was already handled before we got on this call"
- "Since you've already completed everything on your end, we're focused on the next steps"
- Any agent statement that explicitly frames the student as having finished all upstream
  enrollment requirements before this call

NOT qualifying:
- Vague encouragement without explicit completion reference ("you've been doing great")
- Student volunteering completion without agent acknowledgment or confirmation
- Agent referencing one specific completed item without establishing broader completion
- Agent language that implies completion without explicitly stating it

HARD LIMIT — F2_4 IS NEVER COVERED BY THIS RULE
F2_4 (Assessment Re-engaged) requires explicit re-engagement of the assessment as a
priority next step during this call. Blanket prior completion confirmation does NOT satisfy
F2_4 under any circumstances. The assessment must be explicitly addressed regardless of
what was completed before the call.

--------------------------------------------------

BRANCH A — FAFSA NOT YET SUBMITTED (FAFSA_Validated_YN = NO)
Applicable items: F1_1, F1_2, F1_3, F1_4

F1_1 Explicit action: FAFSA status was assessed.
Score 1 ONLY if the agent explicitly asked the student about their FAFSA completion status
at the start of the call.

Acceptable evidence:
- Agent asks directly whether the student completed or submitted the FAFSA form
- Agent references the prior call's FAFSA plan and checks on progress
- Student confirms they have not submitted and agent acknowledges this before proceeding

NOT sufficient:
- Agent assumes FAFSA status without asking
- Agent mentions FAFSA as a future task without assessing current status
- Agent moves directly to FAFSA workflow without establishing current status first

Score 0 if the agent did not explicitly assess FAFSA status before proceeding.

--------------------------------------------------

F1_2 Explicit action: FAFSA workflow was initiated on-call.
Score 1 ONLY if the agent actively guided the student into the FAFSA workflow during the
call AND the student confirmed reaching the login or starting point.

Score 1 ONLY if ALL of the following are present:
1) Agent directed the student to studentaid.gov or the correct FAFSA access path, AND
2) Student confirmed they are logged in or reached the FAFSA dashboard or form starting point

NOT sufficient:
- Agent only described FAFSA without guiding workflow movement
- Agent directed student to FAFSA but student did not confirm reaching the access point
- FAFSA account creation was attempted but reset, interrupted, or unconfirmed
- Partial navigation that stopped before the student reached the FAFSA login or starting point

Score 0 if the student did not explicitly confirm reaching the FAFSA workflow entry point.

--------------------------------------------------

F1_3 Explicit action: Scripted FAFSA handoff was secured.
Score 1 ONLY if ALL of the following are present in the transcript:
1) Agent explicitly stated they cannot remain on the phone while the student completes the
   FAFSA because it is a government-issued form, AND
2) Agent secured a callback within approximately 30 minutes to 1 hour

Scripted language that satisfies this item:
- "I can't stay on the phone with you while you're filling this out since it is a
  government-issued form"
- "I'm going to give you a call back in about 30 minutes to 1 hour"
- Any equivalent language that explicitly states the government form restriction AND commits
  to a near-term specific callback

NOT acceptable for the follow-up timeframe:
- Tomorrow at any time
- Later today without a specific time
- A week or any timeframe beyond same-day
- Any language that does not commit to a near-term specific callback tied to FAFSA completion

Score 0 if either element is missing or the follow-up timeframe does not meet the standard.

--------------------------------------------------

F1_4 Explicit action: Assessment re-engagement was addressed.
Score 1 ONLY if the agent explicitly referenced the assessment requirement as a priority
next step during this call.

Acceptable evidence:
- Agent stated that completing the assessment is an important next step
- Agent asked whether the student has started or completed the assessment
- Agent established a plan or timeline for assessment completion
- Agent used language consistent with the scripted assessment re-engagement

NOT sufficient:
- General mention of program requirements without specific reference to the assessment
- Agent acknowledged the assessment exists without establishing any action or priority

Score 0 if the agent did not explicitly address the assessment as a next step during the call.

--------------------------------------------------

BRANCH B — FAFSA ALREADY SUBMITTED (FAFSA_Validated_YN = YES)
Applicable items: F2_1, F2_2, F2_3, F2_4

F2_1 Explicit action: FAFSA status was assessed and confirmed.
Score 1 ONLY if ONE of the following paths is present:

PATH A — Direct Confirmation
Agent explicitly confirmed the student's FAFSA submission status AND identified the relevant
award year during the call.
Acceptable evidence:
- Agent asked how the FAFSA went and student confirmed submission
- Agent confirmed which FAFSA year was submitted (2526 or 2627 or both)
- Agent referenced the prior submission and confirmed it is on file before proceeding

PATH B — Blanket Prior Completion Confirmation (NEW IN v1.1)
The BLANKET PRIOR COMPLETION CONFIRMATION RULE applies and the agent's language
explicitly establishes all upstream enrollment work — including FAFSA — is complete.
See BLANKET PRIOR COMPLETION CONFIRMATION RULE above for qualifying evidence.

NOT sufficient for either path:
- Agent assumed FAFSA was submitted without explicit confirmation or blanket confirmation
- Agent mentioned FAFSA processing without confirming submission status with the student
- Agent moved directly to next tasks without any acknowledgment of FAFSA completion status

Score 0 if neither Path A nor Path B is satisfied.

--------------------------------------------------

F2_2 Explicit action: GIPI (Government Issued Photo ID) was addressed.
Score 1 ONLY if ONE of the following paths is present:

PATH A — Direct Confirmation
Agent explicitly addressed GIPI status:
- Confirmed GIPI was already uploaded (prior completion credit applies), OR
- Guided the student to upload GIPI during the call, OR
- Established a concrete plan for GIPI upload with a specific timeframe

PATH B — Blanket Prior Completion Confirmation (NEW IN v1.1)
The BLANKET PRIOR COMPLETION CONFIRMATION RULE applies. See above.

PRIOR COMPLETION CREDIT (PATH A)
If the student confirms GIPI was already uploaded and the agent acknowledges this before
pivoting to remaining tasks, score 1. The upload does not need to occur during the call.

NOT sufficient:
- Agent mentioned documents generally without specifically referencing GIPI
- Agent assumed GIPI was uploaded without confirmation
- Agent skipped GIPI without addressing its status

Score 0 if neither path is satisfied.

--------------------------------------------------

F2_3 Explicit action: Transcript was addressed.
Score 1 ONLY if ONE of the following paths is present:

PATH A — Direct Confirmation
Agent explicitly addressed transcript status:
- Confirmed transcript was already submitted (prior completion credit applies), OR
- Guided the student to submit their transcript during the call, OR
- Established a concrete plan for transcript submission with a specific timeframe and source

PATH B — Blanket Prior Completion Confirmation (NEW IN v1.1)
The BLANKET PRIOR COMPLETION CONFIRMATION RULE applies. See above.

PRIOR COMPLETION CREDIT (PATH A)
If the student confirms transcript was already submitted and the agent acknowledges this
before pivoting to remaining tasks, score 1.

NOT sufficient:
- Agent mentioned documents generally without specifically referencing the transcript
- Agent assumed transcript was submitted without confirmation
- Agent skipped transcript without addressing its status

Score 0 if neither path is satisfied.

--------------------------------------------------

F2_4 Explicit action: Assessment re-engagement was addressed.
Score 1 ONLY if the agent explicitly referenced the assessment requirement as a priority
next step during this call.

HARD RULE — BLANKET CONFIRMATION DOES NOT APPLY
F2_4 is never satisfied by blanket prior completion confirmation. The assessment must be
explicitly raised, prioritized, and addressed as a next step in this specific call.

Acceptable evidence:
- Agent stated that completing the assessment is an important next step
- Agent asked whether the student has started or completed the assessment
- Agent established a plan or timeline for assessment completion
- Agent used language consistent with the scripted assessment re-engagement:
  "Now that we have completed your documents, the final task is completing your assessment.
  This is an important part of starting into your program, so let's make this a priority
  so you can get ahead."

NOT sufficient:
- General mention of program requirements without specific reference to the assessment
- Agent acknowledged the assessment exists without establishing any action or priority
- Blanket prior completion language about enrollment tasks

Score 0 if the agent did not explicitly address the assessment as a next step during the call.

--------------------------------------------------

BINARY_ITEMS_APPLICABLE — ALWAYS 4
Regardless of branch, Binary_Items_Applicable = 4 for all Follow-Up calls.
Assessment re-engagement is always included in the denominator.
Do not adjust Binary_Items_Applicable based on degree program or assessment applicability.

--------------------------------------------------

BINARY_ITEMS_MISSED OUTPUT (HARD RULE)
After scoring all applicable binary items, populate Binary_Items_Missed as follows:
- List every applicable item that scored 0 using this exact format: [Item ID] [Short Label]
- Separate multiple items with a semicolon and space
- Output None if all applicable items scored 1
- Do not include items that scored 1

Branch A labels (FAFSA_Validated_YN = NO):
F1_1 FAFSA Status Assessed
F1_2 FAFSA Workflow Initiated
F1_3 FAFSA Handoff Secured
F1_4 Assessment Re-engaged

Branch B labels (FAFSA_Validated_YN = YES):
F2_1 FAFSA Status Confirmed
F2_2 GIPI Addressed
F2_3 Transcript Addressed
F2_4 Assessment Re-engaged

Example: If F2_4 scored 0 on a Branch B call where blanket confirmation covered F2_1–F2_3:
Binary_Items_Missed = F2_4 Assessment Re-engaged
