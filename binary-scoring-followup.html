Binary Compliance Scoring – Follow-Up v1.0

Follow-Up Binary Compliance Scoring
Version 1.0 – Initial Release
RECONSTRUCTION NOTE: This source was not included in the upload batch. It has been reconstructed
from the Follow-Up Binary Compliance Checklist v1.0, Binary Compliance Scoring v1.2 (Interview),
and Follow-Up Behavioral Scoring v1.0. Logic is consistent with all three reference sources.
Confirm against the original file before treating as authoritative.

PURPOSE
Convert Follow-Up binary checklist results into Binary_Compliance_Score (0 to 40 points).
This source applies ONLY when Call_Subtype = Follow-Up.
Do NOT apply this source to Interview calls. Interview calls use Binary Compliance Scoring v1.2.

APPLICABILITY RULE
This source applies ONLY when Call_Subtype = Follow-Up.
If Call_Subtype = Interview, stop. Use Admissions_QPI_Source_Binary_Compliance_Scoring_v1.2.txt instead.

INPUTS REQUIRED
Call_Subtype (must be Follow-Up to apply this source)
FAFSA_Validated_YN (determines which branch was scored in the checklist)
Binary checklist item results (0 or 1) for the applicable branch:
- Branch A (FAFSA_Validated_YN = NO): F1_1, F1_2, F1_3, F1_4
- Branch B (FAFSA_Validated_YN = YES): F2_1, F2_2, F2_3, F2_4

OUTPUT FIELDS
Binary_Items_Met
Binary_Items_Applicable
Binary_Compliance_Rate
Binary_Compliance_Score (0 to 40)

FIXED APPLICABILITY RULE (HARD RULE)
Binary_Items_Applicable = 4 for ALL Follow-Up calls regardless of branch.
Assessment re-engagement (F1_4 or F2_4) is always included in the denominator.
Do not adjust Binary_Items_Applicable based on degree program or assessment applicability.
Do not adjust Binary_Items_Applicable based on FAFSA_Validated_YN branch.

GLOBAL SAFEGUARDS
- No inference allowed.
- Default to 0 for any unclear item.

BINARY PROOF-LOCK (HARD RULE)
- Score a binary checklist item as 1 ONLY when there is explicit transcript evidence for that specific item:
  (a) the agent directly instructs the action, OR
  (b) the agent explicitly confirms completion, OR
  (c) the agent sets a specific time-bound plan and the student verbally agrees.
- If evidence is missing, ambiguous, implied, or based only on prior completion claims without
  verification, score 0.
- Round final score only. Do not round the rate mid-calculation.

CALCULATION

Binary_Items_Met
Count of branch items scored 1.
Minimum: 0 | Maximum: 4

Binary_Items_Applicable
Always 4 for Follow-Up calls.

Binary_Compliance_Rate
Binary_Items_Met / 4

Binary_Compliance_Score
Binary_Compliance_Rate * 40

ROUNDING
Round Binary_Compliance_Score to nearest whole number.
Binary_Compliance_Rate must be reported as a decimal rounded to 2 places.

CAPS
Maximum Binary_Compliance_Score = 40
Minimum Binary_Compliance_Score = 0

SCORE EXAMPLES
All 4 items scored 1: 4/4 = 1.00 * 40 = 40
3 items scored 1:    3/4 = 0.75 * 40 = 30
2 items scored 1:    2/4 = 0.50 * 40 = 20
1 item scored 1:     1/4 = 0.25 * 40 = 10
No items scored 1:   0/4 = 0.00 * 40 = 0

DRIFT CONTROL
- If Binary_Items_Applicable is not 4, something went wrong upstream. Default Binary_Items_Applicable
  to 4 and recalculate.
- Default to the lower score when applicability is unclear.

BINARY_ITEMS_MISSED OUTPUT (HARD RULE)
After scoring all applicable binary items, populate Binary_Items_Missed as follows:
- List every applicable item that scored 0 using this exact format: [Item ID] [Short Label]
- Separate multiple items with a semicolon and space
- Output None if all applicable items scored 1
- Do not include items that scored 1

Branch A labels (FAFSA_Validated_YN = NO):
F1_1 FAFSA Status Assessed
F1_2 FAFSA Workflow Initiated
F1_3 FAFSA Handoff Secured
F1_4 Assessment Re-engaged

Branch B labels (FAFSA_Validated_YN = YES):
F2_1 FAFSA Status Confirmed
F2_2 GIPI Addressed
F2_3 Transcript Addressed
F2_4 Assessment Re-engaged

Example: If F2_2 and F2_4 scored 0 on a Branch B call:
Binary_Items_Missed = F2_2 GIPI Addressed; F2_4 Assessment Re-engaged

OUTPUT VALIDATION
Before finalizing Binary_Compliance_Score:
- Confirm the correct branch was scored (A or B based on FAFSA_Validated_YN)
- Confirm Binary_Items_Applicable = 4
- Confirm Binary_Items_Met = count of items scored 1 in the applicable branch
- Confirm Binary_Compliance_Rate = Binary_Items_Met / 4
- Confirm Binary_Compliance_Score = Binary_Compliance_Rate * 40 rounded to nearest whole number
- If any check fails, recalculate before outputting
