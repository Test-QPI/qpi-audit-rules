Binary Compliance Scoring v1.2

Binary Compliance Scoring
Version 1.2 – Weighted 40 Points, Call-Subtype Aware

PURPOSE
Convert binary checklist results into Binary_Compliance_Score (0 to 40 points).

INPUTS REQUIRED
Call_Subtype
Progression_Ceiling
Stage flags (Stage 1–4)
Binary checklist item results (0/1)

OUTPUT
Binary_Items_Met
Binary_Items_Applicable
Binary_Compliance_Rate
Binary_Compliance_Score (0 to 40)

GLOBAL SAFEGUARDS
- No inference allowed.
- Default to 0 for any unclear item.
BINARY PROOF-LOCK (HARD RULE)
- Score a binary checklist item as 1 ONLY when there is explicit transcript evidence for that specific item:
  (a) the agent directly instructs the action, OR
  (b) the agent explicitly confirms completion, OR
  (c) the agent sets a specific time-bound plan and the student/parent verbally agrees.
- If evidence is missing, ambiguous, implied, or based only on prior completion claims without verification, score 0.
- This rule affects binary item scoring ONLY.
  Do NOT change Stage flags, Starting_Stage, Progression_Ceiling, or applicability based on missing binary evidence.
- Only include items from applicable stages.
- Round final score only (do not round the rate mid-calculation).
- For access/execution items (e.g., FAFSA portal access), the action must be functionally successful. If the transcript shows an outage/maintenance block preventing progress, score 0.

APPLICABLE STAGES DETERMINATION

Authority Rule (HARD)
- Binary stage applicability must follow Admissions_QPI_Source_Binary_Compliance_Checklist_v1.2.txt and Stage Detection output.
- A stage may contribute binary items ONLY if that stage is marked PRESENT = YES by the Stage Detection source.
- Progression_Ceiling helps define the expected workflow range, but it does NOT by itself make a stage binary-applicable if the corresponding stage flag is NO.

Interview Calls
- Include Stage 1 binary items only if Stage_Interview_YN = YES.
- Include Stage 2 binary items only if Stage_Application_YN = YES.
- Include Stage 3 binary items only if Stage_FAFSA_YN = YES.
- Include Stage 4 binary items only if Stage_Documents_YN = YES.

Follow-Up Calls
- Include binary items only for stages marked PRESENT = YES in this call.
- Do not include a stage only because it matches Progression_Ceiling if the corresponding stage flag is NO.

If no later stages are marked YES, score only the stages that are actually present.
If all stage flags are unclear, default to Stage 1 only.

CALCULATION

Binary_Items_Met
Sum of all scored 1 values across applicable stages only.

Binary_Items_Applicable
Count of all checklist items across applicable stages only.

Binary_Compliance_Rate
Binary_Items_Met / Binary_Items_Applicable

Binary_Compliance_Score
Binary_Compliance_Rate * 40

ROUNDING
Round Binary_Compliance_Score to nearest whole number using standard rounding (0.5 rounds up).
Binary_Compliance_Rate should be reported as a decimal rounded to 2 places.

Rounding examples:
- 14/17 = 0.8235 → rate = 0.82 → 0.82 × 40 = 32.8 → Binary_Compliance_Score = 33
- 15/17 = 0.8824 → rate = 0.88 → 0.88 × 40 = 35.2 → Binary_Compliance_Score = 35
- 16/17 = 0.9412 → rate = 0.94 → 0.94 × 40 = 37.6 → Binary_Compliance_Score = 38
- 17/17 = 1.00 → rate = 1.00 → 1.00 × 40 = 40.0 → Binary_Compliance_Score = 40

HARD RULE: Do not truncate. Do not floor. Round to nearest whole number only.

CAPS
Maximum Binary_Compliance_Score = 40
Minimum Binary_Compliance_Score = 0

DRIFT CONTROL
- If Binary_Items_Applicable is 0 (should not occur), set Binary_Compliance_Score = 0 and Binary_Compliance_Rate = 0.00.
- Default to the lower score when applicability is unclear.
